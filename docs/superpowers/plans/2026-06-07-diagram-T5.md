## Goal

Deliver the 375–414px responsive layer (plus a 1023px tablet hinge) for the data-composed
diagnostic diagram, and present the kept Meter/reading as a **tap-to-toggle bottom sheet** on
mobile only. Two deliverables, two owned files:

1. `components/diagram-kit/meter-sheet.tsx` — a mobile wrapper around the kept Meter/reading
   block that adds a `peek | expanded | dismissed` **detent state machine driven by tap-to-toggle**
   (Brandon override: NOT a free-drag sheet), a grabber affordance, a thin "now showing" status
   strip, and passes the C3 verdict signal through **untouched** (the Meter renders red; the sheet
   never re-decides a verdict).
2. `components/diagram-kit/diagram-mobile.css` — ALL mobile `@media` rules: the detent geometry,
   the `>=48px` tap-target floor, `env(safe-area-inset-bottom)` insets, the "tested part stays
   above the sheet at peek" guarantee (targets T4's stable `[data-shape]` root classes), and
   `prefers-reduced-motion` on the sheet transitions.

## Architecture — how it stays at the scalability bar

- **Vocabulary-driven, never per-case.** The sheet keys off `ResolvedScene.shape` (the 10-shape
  `StepShape` union) and `ResolvedScene.focus.selectedPartId` only. Adding a new system/symptom/make
  changes the DATA fed to C3, not this layer — the same detent machine and the same `[data-shape]`
  CSS render every system identically. There is **zero** `if (system === 'fuel')` style branching
  and **zero** `observationMethod`/`kind`/`electricalRole` re-derivation (the show-rule already ran
  in T3).
- **Generic fallback.** An unseen `StepShape` falls through to a default `[data-shape]` rule (no
  per-shape selector matched ⇒ the base mobile layout still applies); the sheet still toggles and
  the focus part still clears the peek edge. Never a blank, never a crash.
- **Honest degrade.** When C3 emits a degraded reading (`gaugeSpec === null`, or verdict
  `'neutral'`), the sheet shows the kept Meter's own "needs field check" / no-number state — the
  sheet adds no fabricated value and colors nothing red. Partial data is the steady state.
- **Leak-lock is upstream.** T5 never adds parts/terminals; it only re-flows what `ResolvedScene.elements`
  already contains. A pressure step has no terminals to show because T3 emitted none.

```tsx
'use client'

/** Mobile bottom-sheet detents for the kept Meter/reading.
 *  peek      = reading band visible, the tested part stays above the sheet edge
 *  expanded  = full EXPECT/NOW + why/source + branch detail
 *  dismissed = sheet off-screen (tech swiped it away; canvas full-height)
 *  Brandon override: tap-to-toggle, NOT a free-drag sheet. */
export type MeterSheetDetent = 'peek' | 'expanded' | 'dismissed'

/** The only inputs are taps + lifecycle, never a drag delta. */
export type MeterSheetAction = 'toggle' | 'dismiss' | 'open'

export function nextDetent(
  current: MeterSheetDetent,
  action: MeterSheetAction,
): MeterSheetDetent {
  switch (action) {
    case 'dismiss':
      return 'dismissed'
    case 'open':
      return 'peek'
    case 'toggle':
      return current === 'expanded' ? 'peek' : 'expanded'
  }
}
```

```tsx
import { useReducer, type ReactNode } from 'react'

/** The tap-driven reducer. No drag delta is ever an input (Brandon override). */
export function useMeterSheetDetent(initial: MeterSheetDetent = 'peek') {
  return useReducer(nextDetent, initial)
}

type MeterSheetProps = {
  /** The kept Meter/reading block, rendered verbatim — T5 does not re-own the gauge. */
  children: ReactNode
  /** Thin "now showing" strip text (scenario · step). NEVER "step N of M", never "AI". */
  nowShowing: string
}

/** Mobile-only bottom sheet around the kept Meter. Desktop renders the Meter inline
 *  (no MeterSheet); the screen (T6) mounts this only at the mobile breakpoint. */
export function MeterSheet({ children, nowShowing }: MeterSheetProps) {
  const [detent, dispatch] = useMeterSheetDetent('peek')

  return (
    <div className="meter-sheet" data-testid="meter-sheet" data-detent={detent}>
      <div className="meter-sheet__status" data-testid="meter-sheet-status">
        Now · {nowShowing}
      </div>
      <button
        type="button"
        className="meter-sheet__grabber"
        aria-label="Toggle reading detail"
        aria-expanded={detent === 'expanded'}
        onClick={() => dispatch('toggle')}
      />
      <button
        type="button"
        className="meter-sheet__close"
        aria-label="Dismiss reading"
        onClick={() => dispatch('dismiss')}
      >
        ✕
      </button>
      <div className="meter-sheet__body">{children}</div>
    </div>
  )
}
```
