# Diagram Rebuild — T2: Figma part kit + export + tokens (owns C2)

**Date:** 2026-06-07 · **Track:** T2 (Wave 1, ships its C2 type-freeze in Wave 0 first) · **Branch base:** `feat/system-data-ingest`

## Goal

Build the diagram's **visual vocabulary once** as deterministic, AI-free React/SVG parts that implement contract **C2** (`components/diagram-kit/part-api.ts`), define the **8 missing design tokens** in `app/globals.css`, and ship a **registry** that resolves a part by DATA (`kind | roleSpecial`) — never a switch. The kit covers the *complete* building-block vocabulary (8 `components.kind`, 6 `electricalRole`, the role-specials, the connection kinds, the test overlays) plus a **generic fallback** so an unseen value renders sanely, never blank. No layout math, no data fetch, no slot logic, no `@xyflow` import, no "AI" string.

## Architecture (where T2 sits)

- **Consumes C1** — the `SystemTopology` types from `lib/diagnostics/load-system-topology.ts` (`TopologyComponent`, `TopologyConnection`, `TopologyPin`, `TopologyScenario`). C2 props are shaped so a `TopologyComponent`/`TopologyConnection`/`TopologyPin` maps onto them **with no casts**. C2 typechecks **with or without** the C1 migration (the new `stepKind`/`isOutOfRange` are optional everywhere they appear).
- **Produces C2** — `part-api.ts` (props + unions + `Terminal`/`PartReading`), `registry.ts` (data-keyed resolver + generic fallback), the part/wire/overlay component library, a kit-catalog preview, and the 8 tokens in `globals.css`.
- **Consumed by T3** (slot-interface / engine — `SlotFill` carries a C2 part-ref + props) and **T4** (templates resolve parts via the registry, drop them in slots). T2 ships C2's **types + typed stubs FIRST (Wave 0)** so T3/T4 compile before art lands.
- **Leak-lock owned here:** `Terminal.visible` is ENGINE-controlled (a prop, defaulting hidden) — parts never render terminals always-on. The structural leak-lock (terminals only on electrical shapes) is enforced in T3/T4; T2's job is to make terminals *impossible to show unless told to*.

## Conventions for this plan

- Test runner is **Vitest** (`pnpm test`). Component render tests use `@testing-library/react` + `vitest`'s `jsdom` env (already a project dep — `tests/unit/*.test.ts(x)` exist). Use `render` + `container.querySelector`/`getByTestId`. Reduced-motion is asserted by stubbing `window.matchMedia`.
- **Cold-cache flake:** first `pnpm test` after a fresh shell can show spurious "PGlite is closed" errors. Rerun once before trusting a failure.
- Every part is a pure SVG-emitting function component. **Inline-JSX SVG only** — no asset import, no build-config change (pipeline decision from tracks.json).
- The fixture is `.design-shots/scene-data.json` (25 components, kinds: pump/sensor/actuator/valve/module/mechanical/splice — note: **no `connector` and no electrical roles** in this snapshot, which is exactly why the registry must cover all 8 kinds + a fallback and never assume the fuel slice).
- Run the **exact** failing test first, see it fail, then the minimal implementation, then PASS, then commit. One task ≈ one commit.

### Step 1.3 — minimal implementation

`components/diagram-kit/part-api.ts`:

```ts
import type { ReactNode } from 'react'

/**
 * C2 — the part-API contract for the diagram kit.
 *
 * Built ONCE against the complete building-block vocabulary. Consumers resolve
 * a part by DATA (kind | roleSpecial) through the registry — never a switch.
 * No system-specific branching lives here or downstream of here.
 */

// --- unions (value arrays the registry + catalog iterate; types derive from them) ---

/** The 8 components.kind values (the data-modeled part vocabulary). */
export const PART_KINDS = [
  'pump', 'sensor', 'actuator', 'valve',
  'module', 'mechanical', 'splice', 'connector',
] as const
export type PartKind = (typeof PART_KINDS)[number]

/** Resolved by role/name, not kind. */
export const PART_ROLE_SPECIALS = ['ground', 'relay', 'fuse', 'power-source'] as const
export type PartRoleSpecial = (typeof PART_ROLE_SPECIALS)[number]

/** The 6 componentConnections.electricalRole values. */
export const WIRE_ROLES = ['12v', 'ground', 'signal', '5v-ref', 'low-ref', 'pwm'] as const
export type WireRole = (typeof WIRE_ROLES)[number]

/** Visual emphasis tier, set by the engine per step (never derived from a system shape). */
export const PART_TIERS = ['focus', 'anchor', 'recede'] as const
export type PartTier = (typeof PART_TIERS)[number]

/** Provenance is expressed by HOW a part draws, not by a badge. */
export const PART_PROVENANCES = ['drafted', 'field-verified', 'needs-field-check'] as const
export type PartProvenance = (typeof PART_PROVENANCES)[number]

/** Which face of the part a terminal sits on (maps onto TopologyPin.edge). */
export type TerminalEdge = 'top' | 'right' | 'bottom' | 'left'

/**
 * A connection point on a component, colored by what it carries.
 * `visible` is ENGINE-controlled — the leak-lock. Terminals are NEVER
 * always-on; a part renders them only when the engine sets visible:true
 * (electrical metering steps only). Defaulting to false at the call site
 * keeps an un-told part terminal-free.
 */
export type Terminal = {
  id: string
  role: WireRole
  edge: TerminalEdge
  label: string
  visible: boolean
  active: boolean
  selected: boolean
}

/**
 * Thin handoff to the EXISTING Meter. C2 does NOT re-own the gauge — this is
 * just the expect/now/verdict triple a part may surface inline. No numeric
 * parsing; verdict is the precomputed signal.
 */
export type PartReading = {
  expect: string
  now: string
  verdict: 'fail' | 'neutral'
}

/**
 * Props every diagram part accepts. `kind` is the wide string from C1
 * (TopologyComponent.kind is `string`); the registry narrows it. stepKind /
 * isOutOfRange are OPTIONAL so C2 typechecks with or without the C1 migration.
 */
export type DiagramPartProps = {
  kind: string
  /** Resolved special role (overrides kind in the registry) when present. */
  roleSpecial?: PartRoleSpecial
  name: string
  tier: PartTier
  active: boolean
  selected: boolean
  provenance: PartProvenance
  terminals: Terminal[]
  reading?: PartReading
  /** Optional — present only after the C1 migration; never required. */
  stepKind?: string | null
  /** Optional pin -> out-of-range map slice; never required. */
  isOutOfRange?: Record<string, boolean>
  onSelect?: () => void
}

/** A registered part component. */
export type DiagramPartComponent = (props: DiagramPartProps) => ReactNode
```

import type {
  DiagramPartComponent,
  PartKind,
  PartRoleSpecial,
} from './part-api'
import { GenericFallbackPart } from './parts/generic-fallback'
import { StubPart } from './stubs/stub-part'

/**
 * Data-keyed part resolution. Consumers pass DATA ({ kind, roleSpecial }) and
 * get a component back — NEVER a switch at the call site, NEVER a per-system
 * branch. roleSpecial (ground/relay/fuse/power-source) wins over kind when set.
 *
 * Wave 0: every key maps to the stub. Wave 1 swaps bespoke parts into these
 * two maps WITHOUT changing resolvePart's signature or the fallback rule.
 */
const KIND_REGISTRY: Record<PartKind, DiagramPartComponent> = {
  pump: StubPart,
  sensor: StubPart,
  actuator: StubPart,
  valve: StubPart,

import { useEffect, useState } from 'react'

/** True when the user has requested reduced motion. SSR-safe (defaults false). */
export function useReducedMotion(): boolean {
  const [reduced, setReduced] = useState(false)
  useEffect(() => {
    if (typeof window === 'undefined' || !window.matchMedia) return
    const mq = window.matchMedia('(prefers-reduced-motion: reduce)')
    setReduced(mq.matches)
    const onChange = () => setReduced(mq.matches)
    mq.addEventListener('change', onChange)
    return () => mq.removeEventListener('change', onChange)
  }, [])
  return reduced
}

import type { ReactNode } from 'react'
import {
  ProbeLead, VoltageDropBracket, AmpClamp, PressureGaugeTee, TestPointMarker,
} from './overlays'

/** The 5 v1 overlays. scope-clip is deferred for v1 (not present). */
export const OVERLAY_KINDS = [
  'probe-lead', 'voltage-drop-bracket', 'amp-clamp', 'pressure-gauge-tee', 'test-point',
] as const
export type OverlayKind = (typeof OVERLAY_KINDS)[number]

const REGISTRY: Record<OverlayKind, () => ReactNode> = {
  'probe-lead': ProbeLead,
  'voltage-drop-bracket': VoltageDropBracket,
  'amp-clamp': AmpClamp,
  'pressure-gauge-tee': PressureGaugeTee,
  'test-point': TestPointMarker,
}

/**
 * Resolve the single overlay for a step by data. An unknown overlay kind
 * degrades to the test-point marker (the most generic hookup) — never blank.
 */
export function resolveOverlay(kind: string): () => ReactNode {
  return (REGISTRY as Record<string, () => ReactNode>)[kind] ?? TestPointMarker
}

import { resolvePart } from '../registry'
import { Wire } from '../wires/wire'
import { resolveOverlay, OVERLAY_KINDS } from '../overlays/overlay-registry'
import {
  PART_KINDS, PART_ROLE_SPECIALS, WIRE_ROLES, PART_PROVENANCES,
} from '../part-api'
import type { PartProvenance } from '../part-api'

function Cell({ label, children }: { label: string; children: React.ReactNode }) {
  return (
    <div data-catalog-cell
      style={{
        border: '1px solid var(--vt-rule)', borderRadius: 8, padding: 12,
        background: 'var(--vt-surface)', display: 'flex',
        flexDirection: 'column', gap: 6, alignItems: 'center',
      }}>
      <svg width={88} height={72} viewBox="0 0 88 72">{children}</svg>
      <span style={{ fontSize: 11, color: 'var(--vt-fg-3)' }}>{label}</span>
    </div>
  )
}

/**
 * Renders the WHOLE kit once: each kind (cycling provenance so all three draws
 * are visible), each role-special, each wire role, each overlay. No data fetch,
 * no slot logic — the visual gate only.
 */
export function KitCatalog() {
  return (
    <div style={{ padding: 16, background: 'var(--vt-bg)' }}>
      <div style={{
        display: 'grid', gap: 12,
        gridTemplateColumns: 'repeat(auto-fill, minmax(120px, 1fr))',
      }}>
        {PART_KINDS.map((kind, i) => {
          const Part = resolvePart({ kind })
          const provenance: PartProvenance = PART_PROVENANCES[i % PART_PROVENANCES.length]
          return (
            <Cell key={kind} label={`${kind} · ${provenance}`}>
              <Part kind={kind} name={kind} tier="focus" active={i === 0}
                selected={false} provenance={provenance} terminals={[]} />
            </Cell>
          )
        })}
        {PART_ROLE_SPECIALS.map((roleSpecial) => {
          const Part = resolvePart({ kind: 'module', roleSpecial })
          return (
            <Cell key={roleSpecial} label={roleSpecial}>
              <Part kind="module" roleSpecial={roleSpecial} name={roleSpecial}
                tier="focus" active={false} selected={false}
                provenance="drafted" terminals={[]} />
            </Cell>
          )
        })}
        {WIRE_ROLES.map((role) => (
          <Cell key={role} label={`wire · ${role}`}>
            <Wire role={role} active x1={8} y1={36} x2={80} y2={36} />
          </Cell>
        ))}
        {OVERLAY_KINDS.map((kind) => {
          const Overlay = resolveOverlay(kind)
          return (
            <Cell key={kind} label={`overlay · ${kind}`}>
              <Overlay />
            </Cell>
          )
        })}
      </div>
    </div>
  )
}
