## Goal

One declarative layout template **per v1 step-shape**, each a `StepTemplate = (scene: ResolvedScene) => ReactNode`. A template takes the already-resolved scene (C3 `ResolvedScene` — slots already filled by T3) and **places each `SlotFill` into a named slot at deterministic geometry** for a fixed canvas, identical for every case and every system. It renders C2 parts (via the C2 registry/fallback) into those slots, emits a **stable root class + `data-shape` attribute** (the responsive seam T5 targets), and threads the `'detail'` content payload into the Meter/sheet region. Layout = pure data + geometry; **zero per-case, zero per-system, zero per-make input, zero AI**.

The leak-lock is **structural, not conditional**: a template's slot-geometry table physically contains no `'overlay'` / terminal-bearing slot keys unless it is the electrical template. A pressure-flow template *cannot* place a `12V`/`GND` terminal because it has no slot for one — there is nothing to branch on.

## Architecture

- **Vocabulary-driven, not shape-by-shape branched.** Each template is a thin component built from ONE shared primitive (`<SlotBox>`) plus a per-shape **geometry table** (a plain const map `SlotName -> SlotPlacement`). The registry maps `StepShape -> StepTemplate`. There is no `switch (system)`, no `if (make === …)`, no `if (observationMethod === …)` anywhere — templates key only on `scene.shape` (already a `StepShape` from C3) via the registry, and on `SlotName` via the geometry table.
- **electrical.ts serves four shapes.** `electrical-probe`, `continuity-ground`, `voltage-drop`, `duty-pwm` all use the SAME template/geometry (source-top / device-under-test-center / ground-bottom + one overlay attachment region). Sub-shape differences are already encoded in the `SlotFill` data T3 produced (which overlay, which wires active) — the template does not re-derive them. This satisfies the DoD: "duty-PWM + voltage-drop degrade onto electrical.tsx without a new file."
- **Generic fallback path is the default, not an exception.** A `StepShape` with no registry entry, or a slot with a `null`/unknown fill, renders the C2 generic fallback into a neutral slot — never blank, never crash. An unseen `SlotFill.part.kind`/`role` is already C2's job (its registry has a fallback); T4 just hands it through.
- **Honest degrade.** A degraded text-only `'route'` arm (fork with `routesToTestActionId === null`) and a `'needs-field-check'` provenance fill both render as quiet, readable boxes via the same `<SlotBox>`. Partial data is the steady state.
- **Geometry is a coordinate table for a fixed virtual canvas** (the proto's world space: a `viewBox`-like 0–1320 × 0–760 stage, the same space `settleCamera` framed). Templates emit absolute slot positions in that space; T4 does NOT compute camera/zoom (that is the assembler/screen's `settleCamera` port — exposed only as a `framing` HINT per template). Per the proto, `settleCamera` framed on the circuit bbox; T4 declares the bbox-relevant slot set per shape via `framing.frameSlots`.
- **No re-declaration of contract types.** Every template imports `StepShape`, `SlotName`, `SlotFill`, `ResolvedScene`, `StepTemplate` from C3 (`@/lib/diagnostics/diagram/slot-interface`) and the part types from C2. T4 defines ONLY template-local geometry types (`SlotPlacement`, `FramingHint`, the per-shape `SlotSet`).

/** Absolute position of a slot on the fixed virtual canvas (proto world space: 1320 x 760). */
export type SlotPlacement = {
  x: number
  y: number
  /** which point of the box sits on (x,y) */
  anchor: 'center' | 'top' | 'bottom' | 'left' | 'right'
  /** recede tier — drives opacity/de-emphasis via templates.css, never display:none */
  tier: PartTier
}

/** A template's ordered named slots and where each sits. The KEYS are the only slots the
 *  shape can ever place — the structural leak-lock (no electrical key => no terminal possible). */
export type SlotSet = Partial<Record<SlotName, SlotPlacement>>

 *  frame; it does NOT compute zoom/pan (that is the screen). */

  /** slots whose bbox the camera frames; empty => frame all placed slots */
  frameSlots: SlotName[]
  /** upper bound on scale, mirroring the proto's per-shape cap (confirm zooms out further) */
  maxScale: number
}

Create `components/diagram-kit/templates/slot-box.tsx`:

```tsx
import type { ReactNode } from 'react'
import type { SlotName, SlotFill } from '@/lib/diagnostics/diagram/slot-interface'
import { resolvePart } from '@/components/diagram-kit/part-api'
import type { SlotPlacement } from './template-local-types'

const ANCHOR_TRANSFORM: Record<SlotPlacement['anchor'], string> = {
  center: 'translate(-50%, -50%)',
  top: 'translate(-50%, 0)',
  bottom: 'translate(-50%, -100%)',
  left: 'translate(0, -50%)',
  right: 'translate(-100%, -50%)',
}

export function SlotBox({
  name,
  placement,
  fill,
}: {
  name: SlotName
  placement: SlotPlacement
  fill: SlotFill | null
}): ReactNode {
  const style = {
    left: `${placement.x}px`,
    top: `${placement.y}px`,
    transform: ANCHOR_TRANSFORM[placement.anchor],
  }
  const common = {
    className: 'slot-box',
    style,
    'data-slot': name,
    'data-tier': placement.tier,
  } as const

  // Honest degrade — never blank, never crash.
  if (fill == null) {
    return (
      <div {...common} className="slot-box is-degraded">
        needs field check
      </div>
    )
  }
  if (fill.kind === 'text') {
    return <div {...common}>{fill.text}</div>
  }
  if (fill.kind === 'part') {
    const Part = resolvePart(fill.part) // C2 registry; returns the generic fallback for unseen kind/role
    return (
      <div {...common}>
        <Part {...fill.props} tier={placement.tier} />
      </div>
    )
  }
  // wire-set / overlay / gauge fills are placed by their owning template region,
  // not as a standalone SlotBox; render the degrade rather than throwing.
  return (
    <div {...common} className="slot-box is-degraded">
      needs field check
    </div>
  )
}
```

> Note: `resolvePart` + the `SlotFill` discriminated union (`kind: 'part' | 'text' | 'wire-set' | 'overlay' | 'gauge'`) are C2/C3 identifiers consumed verbatim. If the C3 freeze names the text arm differently, the only change is the `fill.kind === 'text'` literal — adjust to the frozen tag.

import type { StepShape, StepTemplate } from '@/lib/diagnostics/diagram/slot-interface'
import Confirm from './confirm'
import Electrical from './electrical'
import PressureFlow from './pressure-flow'
import SinglePid from './single-pid'

import Fork from './fork'
import Generic from './generic'

/** Pure map keyed ONLY on the T3-produced StepShape. electrical.tsx serves four shapes:
 *  electrical-probe / continuity-ground / voltage-drop / duty-pwm (sub-type lives in the data). */
export const STEP_TEMPLATES: Record<StepShape, StepTemplate> = {
  confirm: Confirm,
  'electrical-probe': Electrical,
  'continuity-ground': Electrical,
  'voltage-drop': Electrical,
  'duty-pwm': Electrical,
  'single-pid': SinglePid,
  'pressure-flow': PressureFlow,
  'look-inspect': LookInspect,
  locate: Locate,
  fork: Fork,
}

/** Resolve a template by shape; an unmapped/unseen shape falls back to the generic template. */
export function resolveTemplate(shape: StepShape): StepTemplate {
  return STEP_TEMPLATES[shape] ?? Generic
}
