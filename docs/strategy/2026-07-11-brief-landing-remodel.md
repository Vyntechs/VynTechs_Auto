**Date:** 2026-07-11 · **Origin:** Brandon, intake session · **Lane:** marketing surface (independent of Shop OS rows)
**Collision note:** none expected — public marketing routes only; no shared write-paths with rows 23+. Existing branch `chore/landing-copy-rewrite` reviewed for salvage in the annex.

## Goal

A landing page that makes a skeptical shop owner trust PlainWrench within seconds and take one clear next step — original enough that it resembles nothing else ("if it feels familiar to another, it's wrong"), honest enough to survive elite marketing scrutiny, and approved by Brandon **in Figma before any code**.

## What Brandon said (intent)

Total makeover, 100% remodel. Figma must be used; visual-first approval. Psychology throughout: what is the page for, who is it for, why, where's the value. Elite-marketing-team inspection bar. Trust/satisfaction psychology where applicable. Prototype approved before implementation is approved.

## Deliverables (this lane)

1. **Scope:** `2026-07-11-landing-remodel-scope.md` — the page's one job, audience, promise, psychology architecture, required first impressions, design feeling, anti-derivative standard, non-goals, launch bar. Built on the customer-interaction doctrine's personas and trust principles + the plain-language standard's counter voice.
2. **Current-state annex:** `2026-07-11-annex-landing-current-state.md` — what exists today (copy, structure, screenshots), prior GTM exploration worth honoring or retiring.
3. **Figma prototype** — desktop + mobile of the full page, real copy (no lorem), placeholder proof clearly marked PLACEHOLDER. **Delivered v1: https://www.figma.com/design/wsfRi93M41ts5ONQvkGrJZ** — "Landing / Desktop 1440" + "Landing / Mobile 390", positioned per Brandon's mid-session correction as the Shop OS (counter → closeout) with the refuse-to-guess engine as its signature, claims held to shipped rows 1–22.
4. After approval: implementation plan as its own reviewed slice (not this lane).

## Hard gates (in order)

1. **Brandon approves the scope** (or red-lines it — cheap to change here).
2. **Brandon approves the Figma prototype** (visual-first; iterate in Figma until approved).
3. Only then: implementation plan → build → the standard verification chain.

## Scope

- **In:** public marketing surface (landing/home; pricing section if it lives on the page), its copy, structure, visual direction, CTA architecture.
- **Out:** app UI, auth/onboarding flows (CTA target only), pricing *strategy* changes, blog/SEO program, paid-ads program — each its own decision.

## Ethical conversion standard (binding)

No fake urgency, no invented numbers, no fabricated testimonials/logos, no dark patterns. Placeholder proof ships only as clearly-labeled placeholder in the prototype; the live page shows real proof or restructures to not need it. The page may use psychology only the way the product does: show the evidence, tell the truth, make the next step obvious and freely chosen.

## Verify by

- Scope doc answers: page's job, audience, promise, required impressions, what it must never feel like — each specific enough to falsify.
- Figma file link opens; prototype covers desktop 1440 + mobile 390; every section uses real copy in counter voice; zero anti-vocabulary hits; placeholder assets carry a visible PLACEHOLDER tag.
- Brandon's approval recorded on the PR (scope) and in Figma comments (prototype) before any implementation plan is written.

**Stop conditions:** any claim on the page that can't be backed by something real at launch; any pressure to skip the Figma gate.
