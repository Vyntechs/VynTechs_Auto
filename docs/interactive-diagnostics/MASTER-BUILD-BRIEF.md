## §2 — Doctrine (non-negotiable; every PR obeys)
1. **Never guess.** Below 95% confidence, the system says "I'd be guessing" and defers to a human curator. Never fabricate to fill a gap.
2. **Generate, don't copy.** No OEM/licensed manual content, no scraping. Rebuild uncopyrightable facts from training + the tech's own narration + field observation.
3. **Graph, not documents.** Diagnostics are generated by graph traversal, never stored as JSON blobs/rows in a "diagnostics" table. No "merge/stitch" step — connection is the absence of duplication.
4. **AI out of the hot path.** A cache hit renders from pure structured DB reads, no AI call. Orchestration runs only for genuinely novel vehicle+symptom combos.
5. **Cited or it didn't happen.** Every vehicle-specific claim (pin, wire, voltage, spec) must trace to retrieval/corpus/tech input, surfaced as a tappable source.
6. **Role-code wires** by electrical function (5V Ref, Low Ref, Signal, PWM, Ground, CAN-H/L) — never assert OEM wire colors from training; show "—" until a tech captures the real one.
7. **"Not yet captured," not "unknown."** Gaps stay visible as living, fillable fields all the way to the rendered surface.
8. **The bay is the boss.** Phone-first, glanceable, survives greasy hands + bad WiFi + a loud bay, ≤3 questions before it acts.
9. **Honest by design.** The field-capture flywheel uses ethical incentives only — no dark patterns, no fake scarcity, no forced contribution-to-unlock.

## §3 — Architecture (the mental model)
**Build-time: the 4-prompt orchestration** (each a separate API call; never collapse them — collapsing breaks caching/retry/reuse):
- **P1 Research & Prefill** — vehicle (Y/M/M/engine/trans) + system → architecture facts, each tagged TRAINING-CONFIRMED / TRAINING-INFERRED / GAP; refuses to invent part numbers, pins, wire colors. Human vets before it flows on.
- **P2 System Operation Intake** — vetted facts and/or plain-English narration → structured Component nodes + CONNECTS_TO/REPORTS_TO/CONTROLLED_BY edges + an observability profile per component; every inference tagged LAW/LOGIC/PATTERN; ≤10 ranked questions; gaps "not yet captured."
- **P3 Diagnostic Session Generator** — structured model + symptom payload (concern, DTCs, prior results) + 95% gate → scoped minimal circuit slice, candidate failures, a cost/leverage-ordered non-destructive test path with per-step expected reading + source citation + wrong-reading branch + cumulative confidence, and HALTs (contradiction / impossibility / thin-input / observability). Writes TestAction + BranchLogic nodes and Symptom→IMPLICATES_TEST edges.
- **P4a Cross-Vehicle Applicability** — finished diagnostic + candidate vehicles → FULLY/PARTIALLY/NOT/INSUFFICIENT per axis; writes Platform EQUIVALENT_FOR_SYSTEM edges so one diagnostic auto-extends without duplication.

**Storage: typed graph in Postgres** (relational nodes-and-edges; ~12 tables). Components belong to PLATFORMS (not diagnostics) so diagnostics on the same platform share nodes. Node types: Platform, ArchitectureFact, Component, observable_properties, TestAction, BranchLogic, Symptom, TechOutcome, DiagnosticSession. Edge types: BELONGS_TO_PLATFORM, CONNECTS_TO{mode}, REPORTS_TO, CONTROLLED_BY, PROBES, IMPLICATES_TEST{priority}, HAS_BRANCH, EQUIVALENT_FOR_SYSTEM{system}, OUTCOME_OF.

**Runtime serve path:** resolve platform+symptom → load the system-tagged subgraph (components, connections, observable_properties, test_actions+branches; flag tests implicated by the current symptom) → deterministic dagre layout → render on React Flow as a clickable wiring diagram. Pure structured reads; returns null, never throws.

**Knowledge / citation spine:** shop-owner-curated reference cards (pinouts, wiring, theory, locations, bulletins), vehicle-scoped (year-range + make + wildcardable model/engine). The AI may NOT state a vehicle-specific fact unless it first calls a lookup tool; tool returns override training; emits `[ref:item_id]` markers attributed to the active step; source-span verification strips any extracted field whose verbatim quote isn't found in the source. Vision extractors strip *facts* from photographed material, never reproduce the diagram/large OEM text.

**Guided-walk UX:** the topology IS the surface. Scenario picker (key-off/on/idle/load/fault) animates expected behavior; clicking a pin isolates its circuit (others dim) and shows where to back-probe, the live expected reading for that scenario, the range, and "if the reading is wrong" logic. Tests surface implicated-first by invasiveness.

**Compounding loop:** tech outcomes attach to TestAction nodes → after N techs run a test, the node holds the real statistical distribution of measured values → a query surfaces where the field diverged from the AI's expected value and flags it for curator correction. "After enough sessions this becomes what a service manual is supposed to be."

## §5 — Build priorities (decompose into PRs as you judge best)
- **P0 — End-to-end "cold case" flow.** Wire the generation pipeline to a real production caller so a brand-new vehicle+symptom actually flows P1→P2→P3→P4 → graph writes → cache-served render. This is the spine; it's built in pieces but never connected.
- **P1 — The field-capture flywheel (the moat's intake valve).** UX for a tech to capture a wire color/pin/observed value in the bay; it must (a) instantly improve THEIR next identical lookup (own-use payback), and (b) write outcomes back to the shared TestAction nodes. Ethical incentives only (own-use first, give-to-get, status on verified outcomes, exportable ownership).
- **P2 — Curator defer/review gate.** Harden the "never guess" human gate: below-confidence cases route to a curator queue; nothing fabricated reaches a tech.
- **P3 — Eval harness.** A 20–50 case ground-truth set (real diagnoses + confirmed fix) with an accuracy metric, so "never guess" is provable, not a slogan. (Cases supplied by Brandon — see §8.)
- **P4 — Cross-vehicle equivalence.** Activate the dead reuse tables so one diagnostic auto-extends to architecturally-equivalent platforms (P4a edges).
- **P5 — Bay-is-boss UX polish + offline cache.** Phone-first, ≤3 questions, instant cache-served render, PWA offline behavior.

## §6 — Success criteria (how we know it worked — verify each PR against this)
1. A **never-before-seen vehicle+symptom** produces a cited, refusal-gated, interactive topology diagnostic, stored as graph nodes/edges, and on a second request renders from cache with **zero AI calls** (measurable).
2. A tech can **capture a wire color/pin in the bay** and it (a) instantly speeds up their next identical lookup and (b) writes to the shared node visible to the next tech.
3. Below-confidence cases **defer to the curator** and never fabricate; confidence/citation provenance is visible on every step.
4. The **eval set** runs and reports an accuracy number against a stated target (set a starting bar, e.g. ≥X% root-cause in top-3).
5. The same diagnostic **auto-serves an architecturally-equivalent vehicle** via equivalence edges, without regeneration.
6. The whole hot path is verifiably **AI-free on cache hit** (no prompt calls in load).

## §7 — Decisions made (defaults — flip if wrong)
- **Layout:** keep deterministic dagre auto-layout (already shipped). Do not build stored per-system pixel coordinates.
- **Platform:** **PWA web app, phone-first — and it STAYS a PWA for the foreseeable future.** A native iOS/Android app is a ~2030 horizon, explicitly NOT a near-term goal. Do NOT add native-portability complexity, abstractions, or "future native" hedges now — build for the PWA. (The cache-served, AI-out-of-hot-path design already gives the offline-leaning benefits without a native app.)
- **Data:** generate-own + field capture only. **No licensed OEM data, no scraping.** This is both legal and the moat.
- **Scope:** GTM, pricing, and marketing = OUT. Shop management is a separate track governed by `docs/strategy/2026-07-10-shop-os-spec-and-phased-plan.md`; this brief still governs the diagnostic engine.
